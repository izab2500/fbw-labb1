arbetsprocess

Installation av Vite.js

Först installerade jag Vite.js, vilket resulterade i att en filstruktur skapades. Bland dessa filer fanns en .gitignore, som anger vilka filer och mappar som inte ska versionshanteras eller skickas till GitHub.

Skapande av mappstruktur

I mappen src skapade jag två mappar: js och css. I css-mappen har jag en huvudfil som används för allmän styling, samt separata filer för unik styling för varje sida. Detta gör det enkelt att hantera och underhålla de olika stilfilerna (i detta fall var det bara för att öva in det).

Konfiguration av Vite

Jag skapade en vite.config.js-fil där jag gjorde de nödvändiga inställningarna för min multipages HTML-webbplats. Dessutom installerade och konfigurerade jag npm-paketen vite-plugin-image-optimizer och shap för att optimera bilder i formaten AVIF, WebP och JPG vid npm run build.

Versionshantering med Git

Sedan skapade jag en lokal Git-repository med kommandot git init, kopplade den till ett fjärrrepository på GitHub och Netlify. Jag ställde in så att npm install och npm build triggas automatiskt för dist-mappen i mitt projekt för varje "push" till fjärrrepot och main-grenen.

Arbete med Feature-branches

För utvecklingen använde jag feature branches. Jag skapade en ny branch med git checkout -b feature-namn, där jag utvecklade ett element med motsvarande HTML, CSS och JavaScript. Lokala tester genomfördes för att säkerställa att allt fungerade som det skulle innan commit och push till fjärrrepot.


När jag var klar med utvecklingen lade jag till ändringarna till staging med git add . och skapade en commit med git commit -m "Meddelande på imperativ form". Därefter pushade jag till fjärrrepositoryt med git push -u origin feature-namn och skapade en pull request på GitHub. Här beskrev jag vad jag hade gjort, under antagande att andra skulle granska ändringarna.

Uppdatering av Main

Efter att min branch hade mergeats till main på GitHub raderade jag feature-branchen. Lokalt bytte jag tillbaka till main-grenen med git checkout main och raderade feature-branchen med git branch -d feature-namn. För att hålla min lokala main uppdaterad drog jag in de senast ändringarna från fjärrrepositoryt med git pull origin main. Sedan startade jag processen på nytt...

Slutligen

När webbplatsen var färdigställd och all kod hade pushats till GitHub, kontrollerade jag så att allt fungerade lokalt, inklusive responsivitet i de vanligaste webbläsarna. Jag körde sedan npm run build och granskade loggarna. Efter det körde jag npm run preview för att säkerställa att den optimerade versionen av webbplatsen fungerade som avsett. Avslutningsvis testade jag webbplatsen via Netlifys URL.


Sedan skrev jag README.md-fil för att dokumentera projektet och för att lägga till mitt repos URL.
